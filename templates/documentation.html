Getting Started

DataGrabber allows you to extract structured data from various file types including PDFs, images (JPG, PNG), and text files. This guide will walk you through the process of setting up projects and extracting data from each file type.

PDF Documents

Extract data from multi-page PDFs with text or scanned content

Image Files

Process JPG, PNG, and other image formats with OCR technology

Text Files

Extract structured data from plain text documents

Before starting, ensure you have:

  • Created an account and logged in
  • Prepared your documents for processing

Creating a Project

Projects in DataGrabber help you organize and process related documents. Each project can have its own set of fields to extract and can reused multiple times.

1Navigate to Dashboard

After logging in, click projects or you'll be directed to your dashboard. Click on the "Create New Project" button.

The dashboard with the Create New Project button highlighted

2Enter Project Details

Enter a descriptive name for your project that helps you identify its purpose or content and select a folder to store the project files, note that the data that will be extracted using this project template will be automatically stored in this folder as an excel file.

The project details form showing project name, description, and storage location fields

3Define Data Fields

Add the fields you want to extract from your documents. For each field:

  • Enter a field name (e.g., "Invoice Number", "Date", "Total Amount")
  • Provide a detailed description to help the AI understand what to look for
  • You can move fields up and down using the arrow on the right or drag fields up and down using on the left
  • Click "Add Field" to add more fields as needed
Adding custom fields for data extraction with project setup

4Save Your Project

Click "Create Project" to finalize your project setup. You'll be redirected to the project dashboard.

Tips for Defining Fields

  • Be specific in your field descriptions to improve extraction accuracy
  • Include format examples in descriptions (e.g., "Date in MM/DD/YYYY format")
  • For amounts, specify currency and format (e.g., "Total amount in USD with 2 decimal places")
  • For multi-page documents, you can specify where the field appears on a particular page (optional)

Uploading Documents

DataGrabber supports a variety of file formats including PDFs, images (JPG, PNG, WEBP, BMP), and text files. The system can process multiple file types in a single batch.

1Navigate to Your Project

From the dashboard, click on your project name to access the project page.

2Choose Your Upload Method

DataGrabber offers three convenient ways to upload your documents:

  • Click and Select Files: Click the "Upload Documents" button and use the file browser to select one or more files.
  • Drag and Drop: Simply drag files from your computer and drop them into the designated upload area.
  • Add Folder: For larger document sets, click the "Add Folder" button to upload an entire folder of documents at once. Recommended for large batches
The document upload interface showing folder linking and file selection methods
Pro Tip: For large document sets or batch processing, we recommend using the "Add Folder" method for more efficient uploading and processing.

3Mixed File Type Processing

DataGrabber seamlessly handles multiple file formats in a single upload session:

  • Upload PDFs, images, and text files together in one batch
  • The system automatically detects file types and applies the appropriate processing method
  • Each file is processed with the optimal extraction algorithm for its format
  • Results from different file types are standardized into a consistent data format
Upload interface supporting multiple file formats (PDF, JPG, PNG, WebP, BMP, GIF, TXT)

4Processing

After uploading, DataGrabber automatically begins processing your documents:

  • Progress indicators show real-time processing status
  • Each document is analyzed using advanced AI to extract your defined fields
  • Processing time varies based on file type, size, and complexity

5Review Extracted Data

Once processing is complete, you'll see your extracted data in a structured table format, and the data is automatically stored in the folder you created for the project. Review the data for accuracy, you can edit extracted data.

Tips for Document Processing

  • Higher quality documents (clear text, good resolution) yield more accurate results
  • For large batches, organize similar documents in folders for easier uploading
  • Newly extracted data are auto stored in the same excel file on the project folder
  • Documents are processed using your configured AI provider (see AI Settings)
  • For very large document sets, consider processing in manageable batches
  • Supported formats: PDF, JPG, JPEG, PNG, WEBP, BMP, TXT

Automatic Saving

DataGrabber features robust automatic saving capabilities to ensure your work is never lost.

1Project Autosave

All your project configurations are automatically saved as you work:

  • Projects are saved instantly when created
  • Any changes to project settings are saved in real-time
  • Field definitions and extraction templates are preserved automatically
  • Saved projects can be reused for future document processing without reconfiguration
Time-Saving Feature: Create a project once, then reuse it for all similar documents in the future. Your configuration will be ready to use, making batch processing effortless.

2Extracted Data Autosave

All extracted data is automatically saved to your computer:

  • Data is saved locally as soon as extraction is complete
  • All changes and edits to data are saved in real-time
  • Data remains accessible even after closing your browser
  • Your data never leaves your computer, ensuring maximum privacy and security

3Continuous Updates

DataGrabber continuously updates your saved data:

  • Any manual edits to extracted data are saved instantly
  • New documents added to a project are incorporated into the saved dataset
  • Export operations preserve the most up-to-date version of your data
  • No manual save actions required at any point in the workflow

Benefits of Automatic Saving

  • Never lose work due to browser crashes or accidental closures
  • Seamlessly continue work across multiple sessions
  • Build a library of project templates for different document types
  • No need to reconfigure extraction settings for repetitive tasks
  • All data remains private and secure on your local machine

Exporting Extracted Data

Apart from the data auto save, you can also export the extracted data for use in other systems.

1Access Your Project Results

Navigate to your project and view the processed documents and their extracted data.

2Export Data

Click on the "Export" or "Download" button to save your data.

Data export options showing CSV and Excel format choices

3Select Export Format

Choose your preferred format from the available options:

  • Excel (.xlsx) - For spreadsheet analysis and manipulation
  • CSV - For maximum compatibility with other systems

4Save File

Download the file to your computer.

Tips for Data Export

  • Export regularly to ensure you have backups of your extracted data
  • Excel format preserves formatting and is easier to work with for complex data
  • CSV format is better for importing into databases or other systems
  • Check your exported data for any formatting issues before using it in critical systems

Troubleshooting Common Issues

Here are solutions to common problems you might encounter while using DataGrabber.

Issue: Data is not being extracted accurately or fields are missing.

Solutions:

  • Improve your field descriptions to be more specific
  • For PDFs and images, ensure they are high-quality and clear
  • For scanned documents, ensure they are properly aligned
  • Check if text in the document is actually text and not an image

Issue: Unable to upload documents or errors during upload.

Solutions:

  • Check file size (maximum allowed is 20MB per file)
  • Verify file format is supported
  • Ensure you have a stable internet connection
  • Try uploading files one at a time instead of in batch

Issue: Document uploads but fails during processing.

Solutions:

  • Verify the document isn't corrupted or password-protected
  • For large documents, try processing pages in smaller batches
  • Ensure document content is visible and not heavily redacted

Issue: Problems exporting or downloading results.

Solutions:

  • Check if your browser is blocking downloads
  • Try a different export format (Excel vs CSV)
  • For large projects, try exporting in smaller batches
  • Clear browser cache and try again
